# Inbox Integration Test

Tests understanding of Novu's in-app notification inbox component and client-side SDKs.

Pass threshold: 8/10

---

## Scenario 1

Which package should a React developer use for the Inbox component?

- A) `@novu/api`
- B) `@novu/react`
- C) `@novu/js`
- D) `@novu/framework`

**Correct: B**

`@novu/react` provides the React Inbox component, Bell, Notifications, and Preferences components.

---

## Scenario 2

Which package should a Next.js developer use?

- A) `@novu/react`
- B) `@novu/nextjs`
- C) `@novu/next`
- D) `@novu/framework/next`

**Correct: B**

`@novu/nextjs` is optimized for Next.js and handles SSR edge cases. `@novu/framework/next` is for serving workflows server-side, not the inbox.

---

## Scenario 3

How is the HMAC subscriber hash generated?

- A) `sha256(NOVU_APP_ID + subscriberId)` on the client
- B) `hmac-sha256(NOVU_SECRET_KEY, subscriberId)` on the server
- C) `hmac-sha256(subscriberId, NOVU_SECRET_KEY)` on the server
- D) It's automatically generated by the Novu SDK

**Correct: B**

The hash is `HMAC-SHA256` with `NOVU_SECRET_KEY` as the key and `subscriberId` as the message, generated server-side.

---

## Scenario 4

Which identifier is safe to expose in client-side code?

- A) `NOVU_SECRET_KEY`
- B) `applicationIdentifier` (NOVU_APP_ID)
- C) Both are safe for client-side use
- D) Neither — all Novu identifiers must be server-side only

**Correct: B**

`applicationIdentifier` is a public identifier safe for client-side use. `NOVU_SECRET_KEY` must never be exposed to the client.

---

## Scenario 5

A developer integrates the Inbox but no notifications appear. Their workflow sends emails successfully. What's the most likely issue?

- A) The `applicationIdentifier` is wrong
- B) The workflow doesn't have a `step.inApp()` step
- C) The subscriber hash is invalid
- D) WebSocket connection is blocked

**Correct: B**

The Inbox only shows notifications from workflows that include a `step.inApp()` step. Email-only workflows don't appear in the Inbox.

---

## Scenario 6

In a Next.js App Router project, what's required when using the Inbox component?

- A) Server-side rendering with `getServerSideProps`
- B) `"use client"` directive on the component
- C) Dynamic import with `next/dynamic`
- D) Wrapping in a `Suspense` boundary

**Correct: B**

The Inbox is a client-side component. In Next.js App Router, it must be in a file with the `"use client"` directive.

---

## Scenario 7

How does the Inbox receive real-time notification updates?

- A) Polling every 30 seconds
- B) Server-Sent Events (SSE)
- C) WebSocket connection (automatic, no setup needed)
- D) Manual refresh required

**Correct: C**

The Inbox component automatically establishes a WebSocket connection for real-time updates. No additional configuration is needed.

---

## Scenario 8

A developer wants to customize what happens when a notification's primary action button is clicked. How?

- A) Set `onClick` on the action in the workflow definition
- B) Use the `onPrimaryActionClick` prop on the Inbox component
- C) Add a `click` event listener to the Inbox DOM element
- D) Define the handler in the Novu dashboard

**Correct: B**

The Inbox component accepts `onPrimaryActionClick` and `onSecondaryActionClick` callback props.

---

## Scenario 9

A developer wants to build a completely custom notification UI (no pre-built components). Which package should they use?

- A) `@novu/react` with `renderNotification`
- B) `@novu/js` — headless client SDK
- C) `@novu/api` — REST API from the frontend
- D) Build from scratch with WebSocket API

**Correct: B**

`@novu/js` provides a headless client SDK with methods like `novu.notifications.list()`, `novu.notifications.read()`, etc. for building fully custom UIs.

---

## Scenario 10

How do you customize the visual appearance of the Inbox component?

- A) Override CSS classes directly
- B) Use the `appearance` prop with CSS variables
- C) Pass a `theme` prop with a Tailwind config
- D) Create a custom stylesheet and import it

**Correct: B**

The Inbox accepts an `appearance` prop with `variables` for colors, font size, border radius, and other visual properties.
